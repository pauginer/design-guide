Wikimedia Design Guide

Wikimedia is a global movement involving a wide audience. Having design guidelines is useful when designing products of any kind (from t-shirts to mobile applications).

This guide is intended to capture a general design direction that can be useful for different kinds of projects. Designers and developers will be provided with tools to solve general problems to let them focus on the specific needs of their projects.

This is an ongoing work, so it is in evolution. You can always check online the latest released version of the guide. You can help making it grow, see how to contribute below.

How to contribute

You can participate in many ways. The documentation is available in a git repository together with all the design assets you need for your project. You can get all of them, change anything and send the changes back.

Getting the design repository

The design repository is available to download. It contains all the documentation, templates and design resources. Select the "clone or download" button to get the repository (if you have Git installed in your system, cloning is preferred since it allows to contribute back).

Viewing the documentation

The guideline is based on a set of static HTML files for the different sections of the guideline. The menu is added to those pages automatically using Javascript.

Some browsers (such as Chrome) may prevent Javascript to load the table of contents on local files, in that case you'll need to host the files in a local server. Sending changes back

Technically changes can be submitted as usual in git. In terms of coordinating the process we are still working on it…

In order to publish the updated contents to be available live, go to the repo and launch the following command from a terminal: git push -f origin master:gh-pages
